## FEATURE 1 — AUTO-SCROLL (Home carousel only)

Find the "What Clients Say" carousel component on the Home page.

Add this auto-scroll behavior:
- Cards automatically advance every **4000ms (4 seconds)**
- Transition: `transform: translateX(...)` with `transition: transform 500ms ease-in-out`
- After the last card group, loop back seamlessly to first (infinite loop)
- When user clicks left/right arrow manually → clear existing timer → restart fresh 4-second timer
- On `mouseenter` of the carousel container → pause auto-scroll (clearInterval)
- On `mouseleave` of the carousel container → resume auto-scroll (setInterval again)

Implementation pattern (adapt to your framework):
```js
// On mount
autoScrollRef = setInterval(goToNext, 4000);

// On manual arrow click
clearInterval(autoScrollRef);
autoScrollRef = setInterval(goToNext, 4000);

// On mouseenter carousel container
clearInterval(autoScrollRef);

// On mouseleave carousel container
autoScrollRef = setInterval(goToNext, 4000);

// Cleanup on unmount
clearInterval(autoScrollRef);
```

## FEATURE 2 — CARD ENTRANCE STAGGER ANIMATION (Home carousel only)

When a new slide/group of cards comes in (on any transition — auto or manual):

- Each card in the incoming group should animate independently
- Animation: `opacity: 0 → 1` + `translateY(20px → 0px)`
- Duration: `400ms ease-out` per card
- Stagger delay: `0ms, 100ms, 200ms` (i.e., card 1 starts immediately, card 2 after 100ms, card 3 after 200ms)
- This replaces the default "all cards slide in together as a block" behavior

Implementation (React example):
```jsx
// When activeIndex changes, reset and re-trigger animation
// Add a key or animation class to each card based on current slide index
cards.map((card, i) => (
  <div
    key={`${activeIndex}-${i}`}
    style={{
      animationDelay: `${i * 100}ms`,
      animation: 'cardEntrance 400ms ease-out both'
    }}
  >
    ...card content...
  </div>
))

// CSS
@keyframes cardEntrance {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```


## FEATURE 4 — PROGRESS BAR INSTEAD OF DOTS (Home carousel only)

Remove the existing navigation dots (if any). Replace with a thin progress bar:

- Position: centered below the cards, above any existing nav arrows
- Width: ~200px (or same width as dot row was)
- Height: 2px
- Background (track): `rgba(0,0,0,0.15)` or a light gold/neutral color matching site palette
- Fill color: your site's gold/accent color (match existing brand color used elsewhere)
- Behavior: fills from left to right over exactly 4000ms (the auto-scroll interval)
- On manual arrow click → bar resets to 0% and starts filling again from 0
- On hover pause → bar freezes at current fill percentage, resumes from there on mouseleave

Implementation:
```jsx
// Use a CSS animation with a key that resets on each slide change
<div style={{ width: '200px', height: '2px', background: 'rgba(0,0,0,0.12)', borderRadius: '2px' }}>
  <div
    key={slideKey} // increment this on every slide change to restart animation
    style={{
      height: '100%',
      background: 'YOUR_GOLD_COLOR',
      borderRadius: '2px',
      animation: isPaused ? 'none' : 'progressFill 4000ms linear forwards',
      width: isPaused ? currentFillWidth : undefined
    }}
  />
</div>

@keyframes progressFill {
  from { width: 0%; }
  to { width: 100%; }
}
```

Track the fill width using `performance.now()` or a ref when pausing, so it can resume from the exact position.
